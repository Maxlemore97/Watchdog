# Security Policy

## Reporting a Vulnerability

Use GitHub's private vulnerability reporting:
[`Maxlemore97/Watchdog` → Security → Report a vulnerability](https://github.com/Maxlemore97/Watchdog/security/advisories/new).

Please include a reproducer, the affected version, and your assessment of impact.
Expect an initial response within 5 working days.

Do **not** open a public issue or PR with exploit details before a fix ships.

## Threat Model

Watchdog gates AI-mediated package installs and Claude Code plugin loading.
The trust boundary sits between **untrusted artifacts** (published packages,
git-cloned plugins, install commands generated by an LLM) and the **host
machine running the agent**.

### In scope

- **Prompt injection** in fetched artifact files attempting to override the
  analyzer's verdict. Defenses: `<UNTRUSTED>` tag wrapping with
  `html.escape` on path attributes and `</UNTRUSTED` neutralization in
  bodies; deterministic regex prefilter that runs before the LLM; strict
  JSON verdict extraction (fenced or verdict-keyed shallow object only).
- **Malicious install commands** crafted to evade parsing. Defenses:
  `shlex.split` with explicit punctuation chars, recursive subshell
  extraction with depth cap, fail-`ask` on malformed shell.
- **Hostile archive contents** (zip-slip, symlink-out, oversized members).
  Defenses: `tarfile.data_filter` on Python 3.12+, explicit `is_symlink()`
  rejection on extraction and on root manifest reads, per-file and
  per-bundle byte caps, 5 MB download cap.
- **Hostile git repositories** triggering interactive prompts or
  credential leaks during clone. Defenses: `GIT_TERMINAL_PROMPT=0`,
  `GIT_ASKPASS=/bin/true`, `StrictHostKeyChecking=accept-new`,
  `BatchMode=yes`, 20 s timeout, `--depth=1 --filter=blob:none`.
- **Recursive LLM invocation** via hooks triggered from within the
  analyzer's subprocess. Defense: `WATCHDOG_DISABLE=1` set in child env.
- **OSV / registry network failure** masking a vulnerability. Defense:
  fail-closed default (`offline_decision="ask"`), per-call timeouts.
- **DoS via install command fan-out**. Defense: `WATCHDOG_MAX_PACKAGES`
  cap (default 20); larger commands return `ask` without scanning.

### Out of scope

- **Local filesystem integrity.** Verdict caches live under
  `WATCHDOG_CACHE_DIR` (default `~/.cache/watchdog`). A local attacker
  with write access there can plant a forged `allow` verdict. Defense
  belongs to the host's filesystem permissions, not Watchdog.
- **Compromised LLM provider CLIs.** Watchdog shells out to `claude`,
  `gemini`, `openai`, or `ollama`. If the CLI binary itself is
  compromised, Watchdog has no defense.
- **SSRF / private-network probing via `fetch_plugin_git`.** Watchdog
  passes user-provided `https://` / `git@` / `ssh://` URLs to `git
  clone`. Internal-network URLs will attempt to clone and fail; no
  payload is executed, but this is not a hardened network sandbox.
- **Side-channel and timing attacks** against the analyzer cache.
- **Kernel-level attacks** (container escape, ptrace, /proc inspection).

### Known limitations

- **Wheel-only PyPI packages**: no sdist available → metadata-only
  analysis. The LLM still sees authorship/license/url signals but no
  source files. Verdict skews to `ask` in ambiguous cases.
- **Prefilter is regex-based**, not exhaustive. Patterns target
  high-signal indicators (PEM keys, AWS/GitHub/OpenAI/Slack token
  shapes, `curl … | sh`, `env | curl`). Bypass is possible with
  obfuscation; the LLM stage backstops.
- **LLM verdicts are non-deterministic** and cached for 24 h by
  default (`WATCHDOG_LLM_CACHE_TTL`). Switching provider or model
  invalidates the cache.
- **README / docs files** that document `curl … | sh` install patterns
  surface as `ask` rather than `deny` to avoid blocking legitimate
  packages whose READMEs reference common installers. Code/script
  files keep the deny semantics.

## Defense Layers

1. **Parser**: only known package managers + subcommands trigger a scan;
   malformed shell → `ask`.
2. **OSV.dev cross-check**: known CVEs at or above
   `WATCHDOG_MIN_SEVERITY` (default `low`) deny.
3. **Deterministic prefilter**: matches against a small set of
   high-signal patterns; deny on code-file hits, ask on doc-only hits.
4. **LLM source review**: a strict system prompt directs the model to
   treat `<UNTRUSTED>` content as data and return JSON; envelope
   shapes, fenced JSON, and verdict-keyed shallow object are accepted.
5. **Fail-closed defaults**: missing CLI / network failure → `ask`,
   never silent allow.

## Disclosure Policy

We aim to fix critical issues within 14 days and ship a coordinated
release. Reporters credited in the release notes unless they request
anonymity.
